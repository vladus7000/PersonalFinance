# Инструкция для переводчика

Это письмо/файл можно отправить переводчику как есть — программировать не нужно.

## Что за файл

Вам прислали файл вида `app_<код языка>.arb` (например `app_pt.arb` для португальского).
Это обычный текстовый файл в формате JSON, его можно открыть в любом текстовом редакторе.

Внутри — пары `"ключ": "текст"`, например:

```json
"navOverview": "Overview",
```

- `navOverview` — техническое имя строки. **Не трогайте и не переводите его.**
- `"Overview"` — сам текст, который видит пользователь. **Вот это и нужно перевести.**

Итог должен выглядеть так:

```json
"navOverview": "Visão geral",
```

## Что можно и что нельзя

✅ Переводить текст **внутри вторых кавычек** — то, что справа от двоеточия.
✅ Оставлять кавычки, запятые и фигурные скобки `{ }` на месте — это разметка файла, не текст.

❌ Не менять и не переводить часть **слева от двоеточия** (`navOverview`, `appTitle` и т.п.) —
   это технические идентификаторы, приложение по ним находит строку.
❌ Не трогать строки, начинающиеся с `@` (например `"@@locale"`) — это служебная информация,
   пользователь её не видит.
❌ Если в тексте встречается `{что-то}` в фигурных скобках (например `{amount}`) — это место,
   куда приложение подставит число/дату/сумму. Переведите текст вокруг, но саму `{amount}`
   оставьте как есть, символ в символ.

## Если непонятен контекст строки

Некоторые ключи могут быть неочевидны сами по себе (например `moreGoals` — это заголовок пункта
меню, а не глагол «цели»). Если сомневаетесь, что означает конкретная строка или где именно в
приложении она показывается — спросите, не гадайте. Ошибка в контексте хуже, чем присланный вопрос.

## Когда закончили

- Убедитесь, что файл открывается и не «сломан» — если сомневаетесь, вставьте содержимое в любой
  онлайн-JSON-валидатор.
- Не обязательно переводить всё за один раз — можно прислать частично переведённый файл,
  непереведённые строки останутся на английском в приложении, ничего не сломается.
- Пришлите файл обратно тем же именем (`app_<код языка>.arb`).

## Для разработчика: как подготовить и принять перевод

Подготовить файл-заготовку для нового языка (не создавать вручную — скрипт не пропустит ошибки
в структуре JSON):

```bash
dart run tool/scaffold_locale.dart pt   # код языка по ISO 639-1: pt, de, es, uk...
```

Это создаст `lib/core/l10n/app_pt.arb` со всеми текущими ключами и английским текстом-заглушкой —
готовый к отправке файл, ничего в коде менять не нужно (новый язык подключается автоматически,
проверено на практике).

После получения перевода — положить файл на место и проверить:

```bash
flutter gen-l10n
cat lib/core/l10n/untranslated.json   # {} — переведено всё; иначе — список пропущенных ключей
flutter analyze
flutter test
```
